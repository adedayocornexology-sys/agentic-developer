// the curriculum · 6 weeks

A build log, not a lecture series.

Every week ends with something committed to your own repo. You learn by shipping.

Week 0 · Pre-work

Setup & first win

  • Install, auth, IDE integration (VS Code / JetBrains), terminal setup
  • Permission modes explained; your first safe edit
lab Ship one real change to your repo before kickoff
Week 1

The agentic loop & effective prompting

  • How read → edit → run → verify actually works
  • Prompts that get senior-engineer output: specificity, constraints, plan-first
  • Model selection — Opus / Sonnet / Haiku, fast mode, cost awareness
lab Fix three real bugs, plan-first ship A verified fix + your winning prompt pattern
Week 2

Context engineering — the skill nobody teaches

  • CLAUDE.md files: project memory that makes every session smarter
  • Navigating and changing large / unfamiliar codebases
  • Planning mode & extended thinking; managing context on long tasks
lab Write a production CLAUDE.md + run a multi-file refactor ship A merged refactor touching 5+ files
Week 3

Customization & automation

  • Custom slash commands for your recurring workflows
  • Hooks: auto-run formatters, tests, or checks on tool events
  • Skills, settings.json, permissions & a sane allowlist
lab Build 2 slash commands + 1 hook you'll use daily ship A personal automation pack in your repo
Week 4

Extending Claude with MCP & subagents

  • MCP servers: connect Claude Code to your database, docs, GitHub, internal APIs
  • Subagents & parallelization: running independent work concurrently
  • When to delegate vs. do it inline — context hygiene
lab Wire up one MCP server & run a task that uses it ship A working MCP-powered workflow
Week 5

Production workflows

  • Verification-driven development: tests as the contract
  • Clean commits, PRs, review-ready diffs; built-in code & security review
  • Headless / scripted Claude Code in CI
lab Take a feature from issue → PR, fully directed ship An opened PR with passing checks
Week 6 · Capstone

Ship a feature — or build an agent

  • Track A — Ship: take a real feature end-to-end and merge it
  • Track B — Build: create a custom agent with the Claude Agent SDK
  • Live demo day; peer & instructor feedback
ship A shipped feature or a working custom agent
